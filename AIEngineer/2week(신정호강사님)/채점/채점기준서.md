# 2주차 채점기준서 — 신정호 강사님

> **채점 원칙: 문제당 0 or 10, 부분점수 없음**
> 총점 100점 (10문제 × 10점) / 감점 사유는 짧은 키워드로 작성 (20자 내외)

## 채점 방식
- 각 문제: O(10점) / X(0점) 이진 채점
- O: 문제지 요구사항 **모두** 충족 + 하단 검증 코드(assertion) 통과
- X: 하나라도 미충족 / 미제출 / assertion 실패
- 감점 표기: "미제출", 또는 핵심 누락 키워드 (예: `assertion 실패`, `에러메시지 불일치`)

## 주차 주제 / 과제 범위
Transformer/LLM 구현 기초 — Tokenizer, Multi-Head Split, KV Cache, Tool Calling, API 재시도 등 10문항. 각 문제는 TODO 구현 후 하단 Assertion을 통과해야 한다.

## 문제 1: Tokenizer — EOS/BOS 및 Padding 제어

**배점**: 10점
**대상 함수**: `preprocess_sentences(sentences, max_l) -> (input_ids, attention_mask)`

### O (10점) — 모두 충족
- [ ] 각 문장 앞에 BOS(1), 뒤에 EOS(2) 토큰 추가
- [ ] 길이가 `max_l` 미만이면 PAD(0)으로 패딩
- [ ] 반환값이 `(input_ids, attention_mask)` 튜플
- [ ] 두 반환 텐서 모두 `torch.LongTensor` 타입
- [ ] 검증 `preprocess_sentences([[10,20]], 5)[0].tolist() == [[1,10,20,2,0]]` 통과

### X (0점) 사례
- BOS/EOS 토큰 미추가 또는 순서 잘못
- 패딩 값이 0이 아님
- 반환 타입이 LongTensor 아님
- 검증 assertion 실패
- 미제출

## 문제 2: Transformer — Multi-Head Split

**배점**: 10점
**대상 함수**: `split_heads(tensor, num_heads) -> Tensor`

### O (10점) — 모두 충족
- [ ] 입력 `(Batch, Seq, Hidden)` → 출력 `(Batch, Num_Heads, Seq, Head_Dim)`
- [ ] `view`와 `transpose` **둘 다** 사용
- [ ] `head_dim = hidden // num_heads` 로 나눈 결과가 출력 마지막 차원과 일치

### X (0점) 사례
- 출력 차원이 요구 shape과 다름
- view/transpose 외의 방식만 사용 (e.g. reshape + permute만)
- 미제출

## 문제 3: KV Cache — Inference Step Concatenation

**배점**: 10점
**대상 함수**: `append_to_kv_cache(prev_k, new_k) -> Tensor`

### O (10점) — 모두 충족
- [ ] Sequence 차원(dim=-2) 기준으로 `torch.cat` 수행
- [ ] 입력 prev_k(1,1,4,16) + new_k(1,1,1,16) → 출력 shape (1,1,5,16)

### X (0점) 사례
- 다른 차원(dim=0, dim=1 등)으로 concat
- 출력 shape 불일치
- 미제출

**대상 함수**: `get_probs_with_temp(logits, temp) -> Tensor`

- 미제출

## 문제 5: Tool Calling — strict JSON Schema

**배점**: 10점
**대상 변수**: `currency_tool_schema` (dict)

### O (10점) — 모두 충족
- [ ] `type: "object"` 유지
- [ ] `properties`에 `amount`(number), `from_cur`(string), `to_cur`(string, default "USD") 정의
- [ ] `required`에 `amount`, `from_cur` 포함 (to_cur은 선택)
- [ ] `to_cur` 기본값 "USD" 명시

### X (0점) 사례
- 필수 인자 누락 또는 타입 오류
- `to_cur`의 default 누락
- `required` 리스트 누락/오류
- 미제출

**대상 함수**: `parse_model_call(call_str) -> dict`

### O (10점) — 모두 충족

## 문제 7: KV Cache — Multi-Head Attention Cache Update

**배점**: 10점
**대상 함수**: `update_mha_kv_cache(prev_k, new_k_proj, num_heads) -> Tensor`

### O (10점) — 모두 충족
- [ ] `new_k_proj: (Batch, Seq_New, Hidden)` → `(Batch, Num_Heads, Seq_New, Head_Dim)` 변환
- [ ] view + transpose 사용
- [ ] 변환된 new_k와 prev_k를 `torch.cat(dim=-2)`으로 결합
- [ ] 검증: `update_mha_kv_cache(zeros(1,2,4,8), ones(1,1,16), 2).shape == (1,2,5,8)` 통과

### X (0점) 사례
- 차원 변환 누락
- concat 차원 오류
- 검증 shape 불일치
- 미제출

## 문제 8 (킬러): Multi-Step Tool Chaining

**배점**: 10점
**대상 함수**: `multi_step_tool_agent(item, tools) -> str`
**엄격 요구**: 에러 메시지 토씨까지 정확히 일치해야 함

### O (10점) — 모두 충족
- [ ] 3단계 도구 순차 호출: `find_product_id` → `get_discount_rate` → `calculate_price`
- [ ] ID가 None이면 정확히 `"ERROR: PRODUCT_NOT_FOUND"` 반환
- [ ] 할인율이 0.5 초과이면 정확히 `"ERROR: INVALID_DISCOUNT"` 반환
- [ ] 성공 시 포맷 `"Final Price for {item} is {price}"` 반환
- [ ] 검증 `multi_step_tool_agent("Phone", m_tools) == "Final Price for Phone is 400"` 통과
- [ ] 검증 `multi_step_tool_agent("Desk", m_tools) == "ERROR: PRODUCT_NOT_FOUND"` 통과

### X (0점) 사례
- 에러 메시지 문자열 불일치 (띄어쓰기/대소문자 포함)
- 할인율 체크 누락
- 성공 포맷 문자열 불일치
- 미제출

## 문제 9 (킬러): Sliding Window Batch KV Cache

**배점**: 10점
**대상 함수**: `sliding_window_batch_kv(cache, new_val, window_size) -> Tensor`

### O (10점) — 모두 충족
- [ ] 입력 shape `[B, H, S, D]` 지원 (Batch > 1)
- [ ] `window_size` 초과 시 index 0의 값을 버림
- [ ] **for 루프 미사용** — 오직 텐서 슬라이싱/torch.cat만 사용
- [ ] window_size 이하일 때는 단순 concat

### X (0점) 사례
- for 루프 사용
- batch 차원 미지원 (B=1만 동작)
- window_size 초과 시 잘라내는 위치가 잘못됨
- 미제출

## 문제 10 (킬러): LLM API Safety & Retry Orchestrator

**배점**: 10점
**대상 함수**: `call_llm_api_safely(prompt, max_tokens, api_fn) -> dict`
**엄격 요구**: reason 문자열 토씨 정확

### O (10점) — 모두 충족
- [ ] prompt 단어 수 > max_tokens 이면 `"ERROR: CONTEXT_EXCEEDED"` 반환
- [ ] 429 응답 시 최대 2회 추가 재시도 (총 3회)
- [ ] 재시도 사이 `time.sleep(1)` 호출
- [ ] 성공 시 반환: `{"status": "success", "response": "...", "attempts": n}`
- [ ] 3회 모두 429이면: `{"status": "fail", "reason": "RATE_LIMIT_EXCEEDED"}`
- [ ] 기타 에러(500 등) 즉시: `{"status": "fail", "reason": "SERVER_ERROR"}`
- [ ] 검증 `call_llm_api_safely(...).get("reason") == "RATE_LIMIT_EXCEEDED"` 통과

### X (0점) 사례
- reason 키 문자열 불일치
- 재시도 횟수 오류 (총 2회 또는 4회 시도)
- `time.sleep` 누락
- CONTEXT_EXCEEDED 체크 누락
- 미제출
