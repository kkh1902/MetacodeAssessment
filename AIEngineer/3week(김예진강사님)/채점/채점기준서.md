# 3주차 (AIEngineer) LLM API 과제 채점 기준서

> **채점 원칙: 문제당 0 or 10, 부분점수 없음**
> 문제지 원문: `문제지/3주차_LLM_API_문제지.ipynb`
> ※ 정답지 파일이 풀이 파일로 덮어씌워진 상태(2026-04-22 확인). 본 기준서는 **문제지 요구사항만**을 근거로 작성.

## 채점 방식
- 각 문제 **10점** (O=10 / X=0)
- 총점 **100점** (10문항 × 10점)
- 문제지 "코드의 다양성 인정" 원칙 적용 — 요구 기능·출력이 맞으면 구현 방식 자유
- LLM 응답의 비결정성(temperature>0, 샘플링) 때문에 발생하는 모델 추론 오류는 **구현 결함으로 간주하지 않음**. 구현 요구사항(호출·파싱·누적 등) 충족 여부로 판정

## 문제 1: TrackedLLMClient + Observability (10점)

### O (10점) — 모두 충족
- [ ] `PRICE_PER_1K` 또는 동등한 단가 상수 정의 (input `$0.00015/1K`, output `$0.0006/1K`)
- [ ] Exponential backoff 재시도 구현 (최대 3회, 실패 시 예외 전파)
- [ ] 호출별 로그에 **모델·input 토큰·output 토큰·레이턴시·상태** 기록
- [ ] 누적 사용량(총 호출수, 총 input/output 토큰, 총 비용) 추적
- [ ] 3회 호출 후 `get_usage_summary()` 출력 — 누적 값이 정상 반영

### X (0점)
- 재시도 로직 없음 / 로그 필드 누락 / 누적 사용량 집계 오류 / 요약 출력 없음

## 문제 2: Temperature 실험 + 다양성 분석 (10점)

### O (10점) — 모두 충족
- [ ] `temperatures=[0.0, 0.3, 0.7, 1.0, 1.5]` 각각 `n_trials=5` 호출
- [ ] `max_tokens=150` (문제지 예시 주석 준수)
- [ ] **Unique response ratio** = 고유 응답 수 / 전체
- [ ] **Average response length** = 평균 문자 수
- [ ] **Vocabulary diversity** = 고유 단어 / 전체 단어 (응답 합쳐서 계산)
- [ ] 5개 temperature 행의 표 출력

### X (0점)
- 지표 3개 중 하나라도 누락/오계산 / trial 반복 없음 / 표 출력 없음

## 문제 3: Top-p × Temperature 교차 실험 (10점)

## 문제 4: 스트리밍 + 금칙어 필터 + 성능 메트릭 (10점)

### O (10점) — 모두 충족

## 문제 5: 동적 Few-shot + Class Mixing (10점)

### O (10점) — 모두 충족
- [ ] `_compute_similarity`: **Jaccard 유사도**(교집합/합집합, 단어 집합 기준)
- [ ] `_select_top_k`: 유사도 상위 K개 선정
- [ ] `_apply_class_mixing`: 동일 라벨 **3개 연속 방지** 로직
- [ ] `build_prompt`: "다음 리뷰의 감성을 POSITIVE, NEUTRAL, NEGATIVE 중 하나로 분류하세요." 서문 + 예시 블록 + 쿼리 블록 형식 조립
- [ ] 실제 테스트 쿼리로 분류 결과 출력

### X (0점)
- Jaccard가 아닌 임의 지표(단순 substring 매칭 등) / 3연속 방지 누락 / 프롬프트 형식 어긋남

## 문제 6: Stepback Prompting + Judge 비교 (10점)

## 문제 7: CoT + Self-Consistency (10점)

### O (10점) — 모두 충족

### O (10점) — 모두 충족

  - Self-Consistency: N=5, temperature=0.7 (또는 동등 수준)

- [ ] 리더보드 필수 컬럼 모두 포함: `Rank | Strategy | ROUGE-L | Judge | Combined | Cost($) | Quality/$`
