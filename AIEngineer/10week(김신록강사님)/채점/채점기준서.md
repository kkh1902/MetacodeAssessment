# 10주차 채점기준서 — 김신록 강사님

> **채점 원칙: 문제당 0 or 10, 부분점수 없음**
> 총점 100점 (10문제 × 10점)

## 채점 방식
- 각 문제: O(10점) / X(0점)
- O: 문제지 요구사항 모두 충족 (오류 없이 실행 + 출력이 예시와 맥락상 동일)
- X: 하나라도 미충족 / 실행 오류 / 미제출
- 감점 표기: 짧은 키워드(20자 내외)

## 주차 주제
`langchain.agents.create_agent`/`AgentMiddleware` 기반 서브에이전트·핸드오프·스킬즈·라우터 4대 에이전트 패턴 구현 (ModelCallLogger, Command 반환 툴, wrap_model_call, SkillMiddleware, Send 팬아웃 + 결과 종합).

## 문제 5: STEP_CONFIG + apply_step_config 미들웨어
**배점**: 10점
**요구사항**:
- `STEP_CONFIG` dict의 세 단계: `claim_intake`, `damage_assessment`, `compensation_decision`
  - `claim_intake`: tools=[record_claim_type], requires=[]
  - `damage_assessment`: tools=[record_damage_amount], requires=['claim_type']
  - `compensation_decision`: tools=[approve_claim, reject_claim], requires=['claim_type','damage_amount']
- `apply_step_config`에 `@wrap_model_call` 적용
- `request.state.get('current_step', 'claim_intake')`로 단계 조회 → `request.override(system_prompt=..., tools=...)`로 동적 교체
- `requires` 필드가 state에 없으면 `ValueError`
- 출력 포맷 일치
### O (10점) — 모두 충족
- [ ] STEP_CONFIG 세 단계 모두 올바른 tools/requires 구성
- [ ] `@wrap_model_call` 데코레이터 적용
- [ ] `request.override(system_prompt=..., tools=...)` 호출
- [ ] 출력 라인 3개와 포맷 일치
### X (0점) 사례
- 단계 누락 / tools·requires 불일치
- `wrap_model_call` 미사용
- `request.override` 미사용
- 미제출

## 문제 10: 결과 종합 노드 + 전체 워크플로우
**배점**: 10점
**요구사항**:
- `synthesize_results(state)`:
  - `state['results']` 비어 있으면 `{'final_answer': '어떤 에이전트에서도 결과를 찾지 못했습니다.'}` 반환
  - 아니면 각 result를 `**{SOURCE} 조회 결과:**\n{result}` 포맷으로 합쳐 `router_llm.invoke` → `{'final_answer': response.content}`
- `StateGraph(RouterState)` 조립:
  - 노드: `classify`, `product`, `order`, `review`, `synthesize`
  - edge: `START → classify`, `classify → (add_conditional_edges, route_to_agents) → product/order/review`, `product/order/review → synthesize`, `synthesize → END`
- `"노트북 A 스펙이랑 후기 알려주세요."` 쿼리 invoke → `classifications` 분류 결과 + `final_answer` 출력
### O (10점) — 모두 충족
- [ ] 빈 결과 처리 분기 존재
- [ ] `router_llm.invoke`로 종합 응답 생성 → `final_answer` 저장
- [ ] `add_conditional_edges(classify, route_to_agents, ...)`로 팬아웃
- [ ] 세 에이전트 노드 → `synthesize` → END
- [ ] 실행 후 분류 결과 + 최종 답변 출력
### X (0점) 사례
- `add_conditional_edges` 미사용 / 팬아웃 미작동
- `final_answer` 키 누락
- 빈 결과 분기 누락
- 실행 오류 / 미제출
