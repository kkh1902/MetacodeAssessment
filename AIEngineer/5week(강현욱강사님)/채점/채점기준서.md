## 공통 규칙

- 출력 결과 일치 시 정답. 동일 출력 나오는 다양한 접근은 인정
- **킬러 문항(Q3, Q6, Q7, Q9, Q10)**: 코드 외 **실행 결과 스크린샷/출력 캡처** 필수. 없으면 0점
- Q1/Q4 유사도 수치는 소수 4자리 기준 ±0.005 이내 일치
- 사용 스택: `sentence-transformers`, `langchain-google-genai`, `chromadb`, `pdfplumber`, `python-docx`, `beautifulsoup4`, `ragas`

## Q1. 임베딩 & 코사인 유사도 직접 계산 (10점)

**요구사항**
- 모델: `all-MiniLM-L6-v2`
- 코사인 유사도는 **numpy로 직접 구현** (`sklearn` 유사도 함수 금지)
- 유사도 내림차순 순위 출력

**O**: SentenceTransformer 로드 + numpy 구현 + Top-k 순위 정렬 출력 + 수치 ±0.005 일치
**X**: `sklearn.metrics.pairwise.cosine_similarity` 등 내장 함수 사용 / 순위 미출력 / 수치 오차 초과

## Q2. 범용 문서 파서 `parse_document()` 구현 (10점) ★ 고난이도

**요구사항**
- 입력 `str` 경로 → 출력 `List[Document]`
- 형식별 처리:
  - `.pdf`: `pdfplumber`, 10자 미만 시 `[SCAN_PDF_DETECTED] OCR required`
  - `.docx`: `python-docx`, 빈 단락 제거
  - `.html`: `BeautifulSoup`, 태그 정리
  - `.txt`: 노이즈 정리 (반복 헤더/푸터·하이픈 줄바꿈·이중공백)

**O**: 4가지 포맷 모두 분기 + SCAN_PDF 특수 처리 + Document 객체 반환
**X**: 한 포맷이라도 누락 / return 타입이 list[str] / SCAN_PDF 처리 누락

## Q3. Chunking 3개 설정 실험 (10점) ★ 킬러

**요구사항**
- 설정 A/B/C (chunk_size 200/200/500, overlap 0/50/100)
- 각 설정별: 청크 수, 평균 길이, 첫 청크 샘플 출력
- 분석 답변 2개 (overlap 효과, 최적 설정 선택 근거)

**O**: 3개 설정 모두 RecursiveCharacterTextSplitter로 실행 + 수치 출력 + 분석 답변 작성 + 스크린샷
**X**: 설정 누락 / 분석 답변 미작성 / 스크린샷 없음(킬러 0점 조건)
