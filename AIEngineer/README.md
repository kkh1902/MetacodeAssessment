## 주차별 피드백 요약

### 2주차 (신정호) — 난이도 상
- **🎯 비유**: *운전면허 기능시험* — 주차선 하나만 밟아도 실격. 코스(assert)는 명확하지만 합격/불합격이 칼같음.
- **장점**: 엄격한 에러 메시지 형식(`"ERROR: PRODUCT_NOT_FOUND"` 등)과 assert 기반 자동 검증으로 이분성 명확. 실제 프로덕션 시나리오(재시도·캐시·도구 체이닝)를 작은 단위로 분해.
- **개선점**: 부분 점수 없음 → 킬러 문항(8·9·10) 한 줄 실수(예: `time.sleep(1)` 누락)로 0/10 양극화. 검증 코드가 엣지 케이스 일부만 커버함.

### 3주차 (김예진) — 난이도 중상
- **🎯 비유**: *DSLR 수동 모드 촬영 실습* — 셔터·조리개·ISO(Temp·top-p·프롬프트 전략)를 조합해 같은 피사체(문제)의 다른 사진(답변)을 뽑아보는 감각 훈련.
- **장점**: Temperature(0→1.5)·top-p 교차 분석으로 샘플링 공간 시각화. Tree of Thoughts, ReAct 등 **전략 비교** 관점 부각.
- **개선점**: OpenAI API 비용 부담 큼(10문제 × 3회 호출 ≈ $1+). 문제 10 "Quality/$" 리더보드 정렬 기준 모호. Few-shot 예시 부재로 재현성 저하.

### 4주차 (김예진) — 난이도 상
- **🎯 비유**: *대형 도서관 사서 양성 과정* — 책(문서)을 주제 좌표(벡터)로 꽂아두고(임베딩), 손님 질문에 맞는 책 3권을 1초 안에 뽑아(검색), 그 추천이 실제로 쓸 만한지까지 평가(Faithfulness)하는 전 과정.
- **장점**: 임베딩→검색→평가 **전체 RAG 파이프라인**을 하나의 과제에 통합. Hit Rate@K·MRR·Faithfulness 등 평가 지표 다각화.
- **개선점**: OpenAI + SentenceTransformer 양쪽 실행 필수(비용). 비동기 배치 임베딩 검증이 형상 비교에 그쳐 얕음. Faithfulness의 LLM-as-Judge 의존으로 일관성 문제.

### 5주차 (강현욱) — 난이도 상 · **검수 진행 중 (미채점)**
- **🎯 비유**: *RAG 풀스택 종합건강검진* — 임베딩(혈액), 청킹(영양), 검색(혈압), 재정렬(MRI), 평가(종합소견)까지 7개 부위를 한 번에 검사. 어느 한 항목만 빠뜨려도 전체 진단 미완성.
- **장점**: Part 1~7 단원 전부를 10문제로 압축한 **종합 커버리지** — 임베딩→파싱→청킹→검색→재정렬→QA→평가→아키텍처. Q1·Q3·Q6에서 numpy 직접 구현·Recall@k·score_threshold 등 **개념을 수치로 강제**하는 설계가 우수. Q10이 6컴포넌트 클래스 + 데이터 흐름 다이어그램까지 요구해 **추상화 능력**까지 평가.
- **개선점**:
  1. **모델명 검증 완료**: `gemini-3.1-flash-lite-preview` / `gemini-embedding-001` — 2026-04-29 실측 호출 통과 (LLM·임베딩 3072차원 모두 정상). 강사님 지정 그대로 사용 가능
  2. **PRD: LLM 원샷 방어 약함** — Q1·Q4·Q5·Q8은 본인 계정 증빙 없음. Gemini 무료 티어라 본인 API key 캡처가 가능한데도 강제 안 함. 권장: Q8·Q9 출력에 `os.environ['GOOGLE_API_KEY'][:10]` 또는 AI Studio 사용량 대시보드 캡처 추가
  3. **Q9 Faithfulness ≥ 0.5 강제** — Ragas LLM-as-Judge는 Gemini 무료 티어 답변 품질이 낮으면 통과 어려움. 재실행 무한 루프 위험
  4. **Q3·Q10 분석 답변 0점 정책** — 미작성 시 0점이라 명시했지만 분량/깊이 기준 모호 → 채점 일관성 위험
  5. **Q2 PDF 처리** — 별도 샘플 PDF 미제공("로직만 작성")이라 OCR 분기 코드가 실제 검증되지 않음. PDF 1개 동봉 권장

### 6주차 (변선영) — 난이도 중상 · **Claude Code 실측 10/10**
- **🎯 비유**: *미슐랭 심사 + 요리사 자가 점검* — ROUGE는 플레이팅 모양만 보는 심사, BERTScore는 한 입 먹어본 뒤 맛을 평가하는 심사. Self-Correction 루프는 셰프가 손님에게 내가기 전 스스로 간을 보고 다시 만드는 과정.
- **장점**: ROUGE-L(n-gram)과 BERTScore(임베딩) 개념 차이를 **직접 수치로** 체감. Self-Correction 루프로 에이전트 견고성 실습.
- **개선점**: Q5 정답 "112개" 고정 — 모델이 맞출 확률이 낮아 `±5` 허용 범위 필요. Q10 최대 10회 재시도에 조기 종료 조건 없음 → 토큰 낭비. Q1 `top_p: 0.0`은 OpenAI API 유효 범위 밖이라 정정 필요. Q8 테스트 케이스(`alpha=0.1, vocab=3`)는 정답지 로직으로도 기대값 통과 불가 — 파라미터 재설계 필요.

### 7주차 (유형준) — 난이도 상 ⭐ **가장 균형 잡힘**
- **🎯 비유**: *은행 창구 시스템 개발* — 같은 이체를 두 번 받아도 한 번만 처리(멱등성), 위조 신분증 거르고(보안), 모든 거래를 감사 장부(pytest)에 기록. 규정집(JSON-RPC 스펙) 그대로 지켜야 통과되는 금융 IT 감사의 축소판.
- **장점**: 보안(SQL/Prompt/Path Injection), 멱등성(`idempotency_key` 해싱), pytest 커버리지 90% 목표까지 **품질 보증** 의식 형성. MCP 프로토콜을 JSON-RPC 스펙 기반으로 실습.
- **개선점**: Part 2 킬러는 Inspector·curl·pytest 증빙 세트가 필수라 **환경 구성 실패 시 전면 0점**. 제공 데이터(12행)가 작아 대규모 처리 경험 부족.

  5. **Q10 AI 생성 0점 정책**: 검출 방법 미명시. 손그림 사진 의무화 또는 EXIF/draw.io 원본(.drawio) 파일 동봉 강제 필요
  6. **킬러 6문항(Q1·Q3·Q4·Q5·Q7·Q10) 비율 60%**: 한 캡처 누락 시 점수 급락 — 13주차 75.5점 사례와 같은 양극화 위험

### 13주차 (정채진) — 난이도 최상
- **🎯 비유**: *종합 건설 현장 준공 검사* — 설계도(Terraform)·자재 반입(Docker)·배관(Nginx)·전기·소방까지 전부 현장 사진과 함께 제출해야 준공(배포). 도장 하나만 빠져도 준공이 안 나는 감리 최종 단계.
