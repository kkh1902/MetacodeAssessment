# 9주차 채점기준서 — 김신록 강사님

> **채점 원칙: 문제당 0 or 10, 부분점수 없음**
> 총점 100점 (10문제 × 10점)

## 채점 방식
- 각 문제: O(10점) / X(0점)
- O: 문제지 요구사항 모두 충족 (오류 없이 실행 + 출력이 예시와 맥락상 동일)
- X: 하나라도 미충족 / 실행 오류 / 미제출
- 감점 표기: 짧은 키워드(20자 내외)

## 주차 주제
LangChain 기본 모델 호출·구조화 출력, LangGraph `StateGraph`/조건부 라우팅/툴 노드/ReAct 에이전트 및 `InMemorySaver` 기반 멀티턴 대화 구현.

## 문제 6: 조건부 라우팅
**배점**: 10점
**요구사항**:
- `route`: 마지막 메시지 content 길이 ≥ 10 → `"long"`, 미만 → `"short"`
- `long_response`: `AIMessage("긴 질문이에요.")`
- `short_response`: `AIMessage("짧은 질문이에요.")`
- `START`에서 `add_conditional_edges`로 `route` 결과 분기
- 실행 쿼리 `"LangGraph에 대해 자세히 설명해줘"` → `긴 질문이에요.`
- 실행 쿼리 `"안녕"` → `짧은 질문이에요.`
### O (10점) — 모두 충족
- [ ] `route` 함수가 길이 10 기준으로 분기
- [ ] `add_conditional_edges(START, route, ...)` 사용
- [ ] 두 쿼리 실행 결과가 예시와 동일
### X (0점) 사례
- `add_conditional_edges` 미사용 (고정 edge만)
- 분기 기준 오류
- 출력 문구 불일치
- 미제출

- [ ] `tools_by_name` 딕셔너리 구성
- [ ] `tool_node`가 tool_calls 순회 → `ToolMessage` 리스트 반환
- [ ] `"tool_node 테스트 통과"` 출력
### X (0점) 사례

## 문제 9: ReAct 에이전트 전체 구현
**배점**: 10점
**요구사항**:
- `simple_search` tool (`@tool`): Tavily `max_results=2`로 검색, 문자열 반환
- `ReactState`: `messages(add_messages)`
- `llm_call`: LLM + tools 바인딩, `SystemMessage("너는 웹 검색 에이전트야.")` 포함하여 invoke
- `tool_node`: tool_calls 실행 후 `ToolMessage` 반환
- `should_continue`: tool_calls 있으면 `"tool_node"`, 없으면 `END`
- 그래프: `START → llm_call`, `llm_call →(조건부)→ tool_node | END`, `tool_node → llm_call`
- 쿼리 `"2026년 LangChain의 최신 버전을 검색해줘."` 실행 → 마지막 AIMessage content 출력
### O (10점) — 모두 충족
- [ ] Tavily 기반 `simple_search` tool 정의
- [ ] LLM `bind_tools` 적용
- [ ] SystemMessage 포함
- [ ] 조건부 라우팅으로 tool_node ↔ llm_call 루프 구성
- [ ] 실행 성공 및 응답 출력 (검색 결과 반영)
### X (0점) 사례
- Tavily 미사용 / 실제 검색 호출 없음
- `bind_tools` 누락
- 조건부 라우팅 없음 (한 번만 호출)
- 실행 오류 / 미제출

## 문제 10: 멀티턴 대화 + compress 노드
**배점**: 10점
**요구사항**:
- `MultiTurnState`: `messages(add_messages)` + `compressed_result: str`
- `research_node`: 모델 invoke, 응답을 messages에 추가
- `compress_node`: `filter_messages(..., include_types=["ai"])`로 AI 메시지만 추출 → 한 문장 요약하여 `compressed_result` 저장
- `START → research_node → compress_node → END`
- `InMemorySaver` checkpointer, `thread_id="turn-test"`
- Turn 1: `"LangGraph의 주요 특징을 알려줘."`
- Turn 2: `"그 중 가장 중요한 것은 뭐야?"`
- Turn 2 messages 개수 > Turn 1 assert 통과, `"멀티턴 누적 확인"` + compressed 출력
### O (10점) — 모두 충족
- [ ] `filter_messages(include_types=["ai"])` 사용
- [ ] `InMemorySaver` checkpointer 적용 + `thread_id` 동일하게 2회 invoke
- [ ] 요약 결과를 `compressed_result`에 저장
- [ ] assert 통과 (messages 누적)
### X (0점) 사례
- checkpointer 미적용 → messages 누적 안 됨 → assert 실패
- `filter_messages` 미사용 (AI 외 메시지도 포함)
- 단일 턴만 실행
- 미제출
