## LLM 저항 포인트 (Claude가 자주 놓치는 항목)

### Q2
1. **포맷 이원화**: `ratio_in_generation`은 문자열 `"38.80%"`, `ratio_overall`은 float `0.3880`. LLM은 둘 다 float 또는 둘 다 문자열로 통일하기 쉬움.
2. **3단 정렬**: `sort_values(['Generation','count','Type_1'], ascending=[True, False, True])` 정확히 써야 함. LLM은 2단에서 멈추거나 tiebreaker 생략.
3. **Tier 경계**: "1% ≤ ratio_overall < 5% → Common" 반개구간 처리. LLM은 `1 < x < 5` 식으로 경계를 놓치기 쉬움.
4. **shape 검증 힌트** "(N, 6)"에서 컬럼 수 6을 정확히 맞춰야 함. 참고이미지 없으니 LLM은 컬럼 개수를 7 혹은 5로 출력하기 쉬움.

### Q4
1. **함수 내부 print 금지** — LLM은 습관적으로 `print(before)` 넣음 → 스펙 위반.
2. **twinx 사용 강제** — LLM은 대개 두 subplot으로 분리함.
3. **0 division → nan** — LLM 기본 코드는 `0/0 = NaN`인 건 맞지만, `a/0`은 `inf` 반환. 명시적으로 `np.where(before==0, np.nan, ...)` 작성 필요.
4. **docstring 형식** — `"""Args:\n    ...\nReturns:\n    ..."""` 포맷을 정확히 지켜야 함. LLM은 줄바꿈·들여쓰기에서 한두 칸 어긋남.
5. **타입 검증 메시지 문구** — 에러 메시지 문자열이 정확히 일치해야 함.

## 리스크

- **Q4의 twinx + grouped bar 조합**은 plot 레이아웃이 겹치기 쉬움. 답지에서 `ax2.grid(False)`, 범례 수동 병합 등 주의 필요.
- "docstring 형식"을 지나치게 엄격히 채점하면 학생 반발 가능. **채점 기준을 "핵심 섹션(Args/Returns) 존재 여부"로 완화** 권장.
- 참고이미지 제거로 **시각화 출력 기대치 모호**해짐. 출력 예시 문구 (x축/y축 라벨, 제목)만 텍스트로 명시 권장.
