## LLM 저항 포인트

### Q3
1. **GridSpec 비율 계산**: LLM은 `subplots(2,2)`로 안이하게 풀거나, GridSpec 쓰되 비율을 1:1:1 균등으로 두어 레이아웃 어긋남.
2. **Figure-level 범례**: 각 subplot에 개별 범례를 두는 실수 빈번. "`fig.legend()` 하나만 사용" 명시.
3. **Heatmap annotation**: 셀 안에 숫자 표기 여부 — D안에서는 "정수 값 표기, 값이 0이면 공백" 명시.
4. **Savefig 옵션**: `dpi=150`, `bbox_inches='tight'` 정확히. LLM은 기본값으로 저장.
5. **Type_2 NaN 처리**: `Type_2`가 결측이면 heatmap에서 제외 또는 "None" 카테고리로 통합. 스펙에서 "None 포함, 라벨은 'None'"으로 명시 → LLM 종종 dropna해버림.

### Q6
1. **executemany 파라미터 형식**: `(?,?,?,?)` vs `(%s,%s,%s,%s)` — 커넥터에 따라 다름. 스펙에 커넥터 지정 필수.
2. **인덱스 생성/삭제 순서**: (a) 테이블 비우기 → 인덱스 없는 상태 INSERT → 시간 측정 → (b) `TRUNCATE` 후 인덱스 생성 → INSERT → 시간 측정. 이 순서 틀리면 측정값 무의미. LLM은 이 순서를 혼동하기 쉬움.
3. **time.perf_counter vs time.time**: LLM은 보통 time.time을 씀. 정밀도가 필요하므로 `perf_counter` 지정.
4. **`cursor.executemany`의 배치 크기**: 10k row 이상일 때 드라이버에 따라 chunked 필요. 스펙에 "한 번에 모든 row 전달" vs "1000건씩 chunked" 명시.

## 리스크

- **커리큘럼 외 스킬**: GridSpec, executemany, 인덱스 성능 측정은 전형적인 11주차 DA 강의 범위 밖. **선행 강의 없으면 학생 혼란 큼**. 도입 전에 강의자료 보강 필수.
