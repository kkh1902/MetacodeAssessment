# 설계안 B — 도메인 규칙 삽입 (Q2 + Q4)

## 요약

포켓몬 세계관의 **임의 비즈니스 규칙**(속성 상성표, 전투력 가중 공식 등)을 문제 안에 명시해, 학생은 표대로 계산만 하면 되지만 LLM은 "왜 이런 규칙?"을 일반 지식으로 추론하지 못해 실수를 유발. 학생 몰입도를 높이면서 LLM 저항을 확보.

---

## 변경 전/후 비교

### Q2 — 속성 상성 기반 승률 계산

| 항목 | v1 | v2 (B안) |
|---|---|---|
| 주제 | type별 count + ratio | type별 **가상 상성표** 기반 승률 계산 |
| 규칙 | 없음 | 문제지에 상성표 제공: `Fire > Grass`, `Water > Fire`, `Grass > Water`, 그 외 조합은 무승부 |
| 계산 | count, ratio 1자리 | 각 type에 대해 "전체 type과 1:1 매치 시 승수", "무승부수", "패수", "승률(%)" 계산 |
| 출력 | 2컬럼 DataFrame | 5컬럼 DataFrame: `Type_1, wins, draws, losses, win_rate` |
| 정렬 | count 내림차순 | win_rate 내림차순 → type 알파벳 오름차순 |

**상성표 예시 (문제지 내 코드 블록으로 제공)**:
```python
advantage = {
    'Fire':  ['Grass', 'Ice', 'Bug', 'Steel'],
    'Water': ['Fire', 'Rock', 'Ground'],
    'Grass': ['Water', 'Rock', 'Ground'],
    'Electric': ['Water', 'Flying'],
    # ...
}
# 규칙: A → B가 advantage[A]에 있으면 A 승. 양방향 모두 없으면 무승부.
```

### Q4 — 전투력 지수 계산 및 시각화

| 항목 | v1 | v2 (B안) |
|---|---|---|
| 주제 | 값 제거 함수 + before/after 막대 | **전투력 지수(BI) 계산 함수** + 세대별 상위 포켓몬 시각화 |
| 공식 | 단순 count | `BI = 0.4·Attack + 0.3·Defense + 0.2·Sp.Atk + 0.1·Sp.Def`, Legendary는 `× 1.3` |
| 시각화 | side-by-side bar | 세대별 **boxplot** + 각 세대 top 3 포켓몬 annotation (이름·BI 표기) |
| 함수 스펙 | 값 제거 | 입력: df, 가중치 dict, legendary 배수. 반환: BI 컬럼 추가된 새 df |

**문제지 제시 공식 블록**:
```
BI(pokemon) = Σ (stat_value × weight) × legendary_multiplier
weights = {'Attack': 0.4, 'Defense': 0.3, 'Sp. Atk': 0.2, 'Sp. Def': 0.1}
legendary_multiplier = 1.3 if Legendary==True else 1.0
```

## LLM 저항 포인트

### Q2
1. **상성표 해석 오류**: LLM은 "Fire > Grass"를 "Fire가 Grass보다 많다"로 해석하거나, advantage dict를 역방향으로 오인 가능.
2. **무승부 조건**: "양쪽 모두 advantage에 없으면 draw" 명시. LLM은 한쪽에만 있어도 draw로 간주하는 실수 빈번.
3. **Self-match 제외**: "같은 type끼리는 매치 대상에서 제외" 규칙 추가. LLM 종종 자기 자신 포함해 승수 계산.
4. **win_rate 분모**: "wins / (wins + losses)" 인지, "wins / total_matches"인지 명시 필요. B안에서는 후자 지정.

### Q4
1. **컬럼명 공백 처리**: `Sp. Atk`, `Sp. Def`는 공백 포함. LLM은 종종 `Sp_Atk`로 리네이밍해서 KeyError.
2. **Legendary 적용 순서**: "가중합 후 배수" 순서. LLM은 가끔 stat별로 배수 적용 후 합산 → 결과 동일하지만 배수 위치가 다르면 의도 다른 출력.
3. **Annotation 좌표**: top 3 포켓몬을 boxplot 위에 텍스트로 표시. LLM은 matplotlib `ax.annotate`의 좌표계(data vs axes fraction)를 혼동.
4. **함수 시그니처**: 가중치와 배수를 모두 인자로 받도록 강제 → hardcoding 유도 방지.

## 리스크

- **난이도 급상승 가능성**: Q2+Q4 모두 B 적용 시 Medium→Hard 점프가 커서 낙오 학생 발생 가능. 한 문제만 B, 나머지는 A로 완충 권장.

- **Boxplot + annotation**은 matplotlib 경험 적은 학생에게 벽이 될 수 있음. annotation 요구를 "top 3 이름만 subtitle에 표기" 수준으로 완화 가능.
