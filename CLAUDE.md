# MetacodeAssessment 채점 가이드

## 채점결과.xlsx 표준 형식

**템플릿 파일**: `양식/채점결과_양식.xlsx` — 새 주차를 채점할 때 이 파일을 복사해서 시작한다. 헤더·총점/피드백 행·스타일이 미리 세팅되어 있다.

모든 주차의 `채점결과.xlsx`는 다음 매트릭스 형식을 따른다.

### 시트 구조

| 행 | 내용 |
|----|------|
| 1행 | 헤더: `문제` + 학생 이름들 (총점 낮은 순 → 높은 순, 또는 높은 순 → 낮은 순 일관성 유지) |
| 2~11행 | `Q1` ~ `Q10` (각 문제별 결과) |
| 12행 | `총점` |
| 13행 | `복붙용 피드백` |

### 각 셀 규칙 (DA 9주차 · 12주차 표준)

- **Q1~Q10 행**: 정답이면 `O`, 오답이면 **짧은 감점 사유**만 기재
  - ⚠️ **문제 설명·재진술 금지**: 무엇이 왜 틀렸는지 **핵심 키워드**만 (예: `BeautifulSoup 미사용`, `head(10) 미사용`)
  - ⚠️ 괄호 안 부연 설명 금지 — `S3 업로드 캡쳐본 미제출 (로그 텍스트만 존재)` ❌ → `S3 업로드 캡쳐 없음` ✅
  - 사유 길이는 **한 줄, 20자 내외** 지향
  - 단순 `X` 금지 — 반드시 사유 명시
  - 미제출은 `미제출` / `미작성`으로 표기
- **총점 행**: O 하나당 10점, 총 100점 만점의 합계 숫자만 기재
- **복붙용 피드백 행**:
  - 만점자: `100` (단일 숫자, "(감점 없음)" 같은 부가 텍스트 금지)
  - 감점자: `{총점}\n{Q번호}. {사유}\n{Q번호}. {사유}...` (줄바꿈 `\n`으로 감점 항목 나열)
  - 예: `90\n3. 검색 결과 미반영(홈 인기글 수집), 컬럼명 '정보' 오류`
  - **셀 사이즈를 콘텐츠 길이에 맞게 자동 조정 (셀 안에서 모든 텍스트가 보이도록)**:
    - `wrap_text=True` 적용하여 자동 줄바꿈
    - **열 너비**: 학생 열은 콘텐츠 한 줄의 평균 길이 기반 30~80자 사이로 설정 (한글 글자는 2 char width로 계산)
    - **13행(복붙용 피드백) 행 높이**: 셀 내부에서 가장 긴 텍스트의 줄 수 × 16pt 기준으로 자동 설정 — 모든 학생 열을 보고 가장 긴 셀에 맞추되, `\n` 분할 후 각 줄이 wrap되어 차지하는 줄 수까지 합산
    - 1열(문제) 너비는 14자 정도로 고정
    - xlsx 저장 후 Excel/LibreOffice에서 열었을 때 잘림·`#####` 없이 모든 사유가 가시 노출되어야 함
    - openpyxl로 일괄 처리: `ws.column_dimensions[col].width`, `ws.row_dimensions[13].height`, `Alignment(wrap_text=True, vertical='top')`

### 예시 (DA 9주차 · DE 5주차 샘플)

```
문제           정경우   김윤서   황성욱                               공성수
Q1            O        O        O                                    driver.quit 호출
Q2            O        O        O                                    BeautifulSoup 미사용
Q3            O        O        검색 결과 미반영(홈 인기글 수집)       head(10) 미사용
...
Q10           O        O        O                                    미제출
총점          100      100      90                                   0
복붙용 피드백  100       100      90                                   0
              (감점     (감점    3. 검색 결과 미반영...                  1. driver.quit 호출
              없음)     없음)                                          2. BeautifulSoup 미사용
                                                                     ...
```

### 셀 스타일 (채점기준서 기준)

| 영역 | 배경 | 글씨 | 정렬 |
|------|------|------|------|
| O 셀 | `C6EFCE` (연초록) | `006100` (진초록) | 가운데 |
| X 셀 (감점 사유) | `FFC7CE` (연빨강) | `9C0006` (진빨강) | 좌측 |
| 헤더 | `4F81BD` (파랑) | 흰색 | 가운데 |
| 총점 | `305496` (진파랑) | 흰색 | 가운데 |
| 피드백 (만점/90+) | `FFF2CC` (연노랑) | 검정 | 좌측 |
| 피드백 (80 이하) | `F4CCCC` (연한 빨강 3) | 검정 | 좌측 |

### 정렬

- 학생 열 정렬: **총점 내림차순** (좌→우) 또는 기존 파일의 정렬 방식 유지
- 동점자는 이름 가나다순

### 부가 산출물

**채점 결과물은 `채점결과.xlsx` 하나로 통일한다.** 아래 파일은 생성·보관하지 않는다:
- ❌ `{N}주차_채점결과.xlsx` (접두어 있는 복제본)
- ❌ `채점결과.csv` (xlsx와 중복)
- ❌ `채점결과.py` (엑셀 생성 스크립트 — 1회성으로 사용 후 삭제)
- ❌ `{N}주차_채점_리포트.md` / `채점상세.md` (xlsx 복붙용 피드백으로 충분, 서술 리포트 불필요)

동일 주차 디렉토리에는 다음 2개 파일만 둔다:
- `채점결과.xlsx` — 최종 채점 결과 (위 양식, `양식/채점결과_양식.xlsx`에서 복제하여 작성)
- `채점기준서.md` — 각 문제의 O/X 판정 기준 (문제지 기반 작성)

## 채점 원칙

- **O/X 이진 채점**: 각 문제 10점, 총 100점
- 감점 시 사유는 **코드/파일 기준으로 구체적으로** 명시 (추측성 피드백 금지)
- 제출 형식(docx/pages/rtf/pdf 단독)만으로 0점 처리하지 않고 **실제 코드 내용으로 판정**
- 주차별 세부 기준은 `{주차}/채점기준서.md`와 `{주차}/CLAUDE.md`를 따른다

## 강사님 제출 자료 요구사항

### 정답지 ipynb — 실행 결과(cell outputs) 동봉 필수

강사님께 정답지를 수령할 때 **반드시 실행 결과가 포함된 상태**로 받아야 한다.

**요구 이유**:
- 채점자가 학생 풀이와 **출력 수치·응답 원문을 직접 대조** 가능
- 정답이 다양한 문제(프롬프트 엔지니어링·서술형 등)에서 **"강사가 본 정답 출력"**이 판정 기준의 기본점이 됨
- 실행 안 된 정답지는 코드 틀린 곳을 채점자가 대신 디버깅해야 함 → 채점 효율 하락

**체크리스트** (정답지 수령 시 확인):
- [ ] 모든 셀에 `outputs` 필드가 비어있지 않음 (print·응답·수치 보존)
- [ ] `execution_count` 가 순차적으로 찍혀있음 (셀 실행 순서 확인 가능)
- [ ] 에러 없이 끝까지 실행된 상태 (마지막 셀까지 output 존재)
- [ ] 랜덤성 있는 문제(LLM 응답 등)는 **강사님이 직접 실행한 응답 원문**이 남아 있음
- [ ] 외부 API/라이브러리 버전 정보가 확인 가능 (필요 시 `pip freeze` 동봉)

**미충족 시 조치**:
- 코드만 있고 출력 없음 → 강사님께 재요청 (실행 후 재제출)
- 일부 셀만 실행됨 → 누락 셀 실행 요청
- 에러 상태로 중단됨 → 환경 정보와 함께 재실행 요청

**카톡/메일 요청 문구 예시**:
```
정답지 ipynb 파일을 실행 결과(cell outputs)까지 포함된 상태로 
보내주시면 감사하겠습니다.
→ 채점 대조 시 강사님이 본 응답 원문과 수치를 기준으로 판정합니다.
```

### 문제지 ipynb

- 학생용이므로 **실행 결과는 비어있어야 함** (`outputs: []`, `execution_count: null`)
- 기대 출력은 "출력 예시" 주석으로 명시
- AI 생성 흔적(과도한 이모지·장황한 설명) 정리 요청 가능

### 학생 풀이

- `풀이/` 폴더에 실제 수강생 제출물만 포함
- 정답지 사본이 실수로 들어간 경우 즉시 제거 (AIEngineer 4주차 사례)
- 사용자(채점 담당) 본인 풀이는 `내풀이_v{N}.ipynb`로 구분

## 채점기준서 작성 원칙

- **채점기준서는 반드시 `문제지`를 기반으로 작성한다.**
  - 문제지에 명시된 요구사항(함수명, 컬럼명, 파일명, 출력 형식, 사용 라이브러리 등)을 그대로 O/X 판정 기준으로 변환
  - 문제지에 없는 요구사항을 임의로 추가하지 않는다 (예: 문제지에 없는 `display()` 강제 요구 금지)
- **정답지(답지)는 참조용으로만 사용한다.**
  - 답지의 특정 구현 방식을 유일한 정답으로 강제하지 않는다
  - 답지와 다른 접근이라도 문제지 요구사항을 충족하면 O
  - 답지는 "예상 출력 형태" / "일반적인 해결 패턴" 확인 용도로만 활용
- 따라서 `채점기준서.md` 각 문제 항목에는:
  1. **문제지 요구사항** (필수 충족 조건)
  2. **O 판정 기준** (문제지 기반)
  3. **X 판정 사례** (문제지 요구사항 미충족 케이스)
  4. (선택) 참조용 답지 경로

## 문제 설계 원칙 — LLM 원샷 방어 (PRD)

**🏆 문제 평가 1순위 기준**: 모든 과제 문제는 **Claude Code / ChatGPT 등 LLM으로 원샷 풀이가 불가능해야 한다.**

> **문제 수준·실무 완성도·난이도보다 우선하는 평가 기준.** 문제가 아무리 실무적이고 고난이도여도 LLM 원샷으로 풀리면 변별력 0. 반대로 문제가 단순해도 본인 수행이 구조적으로 강제되면 좋은 문제다.

**평가 우선순위**:
1. 🏆 **LLM 원샷 방어** (본인 수행이 구조적으로 강제되는가)
2. 문제 완성도·난이도·실무성
3. 채점 효율성 (자동 채점 여부, 판정 명확성)
4. 정답 일관성 (답이 다양한 경우 채점 기준 명시)

### 왜 필요한가

- LLM이 코드·서술형·분석을 전부 원샷 생성 가능 → 채점자가 본인 수행 여부 판별 불가
- `ipynb outputs` 보존만으로는 방어 불가 (LLM이 Colab에서 실행 후 출력까지 포함 생성 가능)
- 따라서 문제 자체에 **"LLM이 대신할 수 없는 사람 에러 포인트"** 가 포함되어야 함

### 사람 에러 포인트 4계층

1. **본인 계정 필수 산출물** (가장 강력, Colab 써도 뚫리지 않음)
   - LangSmith trace URL (본인 조직/계정 ID 포함 URL)
   - OpenAI / Anthropic 사용량 대시보드 (제출일 토큰 사용 기록)
   - HuggingFace `whoami` 출력 (본인 계정명)
   - GitHub repo commit (본인 GitHub 계정)

2. **실시간성 (타임스탬프)**
   - `datetime.now()` 출력이 제출 주간과 일치
   - Tavily / 웹 검색 결과 URL이 제출 주간 기사

3. **개인화된 입력·출력**
   - 본인 이름·기수를 프롬프트/응답에 포함
   - 본인 관심 주제로 추가 실행 1회

4. **외부 도구 연동 증빙 + 로컬 실행 증명**
   - MCP Inspector 연결 화면 (유형준 7주차 표준)
   - Docker/ECS/GCP 콘솔 캡처 (정채진 13주차 표준)
   - 캡처 시 **`date && hostname` 출력 포함** (유형준 방식)
   - 중간 결과/로그/diff 제출 필수, 최종 결과만 제출 시 0점

### Colab 대응

- ❌ **뚫리는 증빙**: `nvidia-smi`, `pip install` 로그, 로컬 파일 경로 — Colab이 동일하게 제공
- ✅ **뚫리지 않는 증빙**: 본인 계정 로그인 필요한 것 (OpenAI/LangSmith/HuggingFace/GitHub)
- Colab 사용 시에도 **Colab notebook 공유 URL**은 본인 Google 계정 증빙으로 활용 가능
