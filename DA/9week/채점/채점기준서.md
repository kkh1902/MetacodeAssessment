## Q6. SQL 쿼리 (10점)

**만점 조건: Q6-1과 Q6-2 둘 다 충족해야 10점, 하나라도 틀리면 0점**

## Q9. Git - Request-commit + B-commit (10점)

**만점 조건**
- A-Branch의 작업을 `Request-commit`으로 main에 Merge
  - `공유-메모.txt`, `A-memo.txt` 내용 수정 (지정 내용 반영)
- `B-Branch` 생성 후 `B-commit` 수행
  - `공유-메모.txt` 내용 수정 (B 작업사항 추가)
  - `B-memo.txt` 파일 생성 및 지정 내용 입력
- 브랜치명·커밋 메시지·파일 내용 PDF와 동일

## Q10. Git - Merge + finish-commit (10점)

**만점 조건**
- B-Branch를 main에 Merge하며 충돌 해결
- 충돌은 **최신 작업 상태 반영** 방향으로 해결
- `finish-commit` 수행 (merge commit, parents 2개)
- **핵심 5개 커밋**(`first-commit`, `A-commit`, `Request-commit`, `B-commit`, `finish-commit`)이 존재하고 순서가 맞을 것
  - 인코딩 수정 등 부수적 추가 커밋은 허용
  - 단, **핵심 커밋을 임의로 쪼개거나 중복 생성**(예: `first-commit`을 두 번 만들기, 불필요한 `master-final-check` 추가 등)한 경우는 오답
- GitHub Repository에 push되어 확인 가능한 상태
