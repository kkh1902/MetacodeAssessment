### Q2. 포켓몬 타입별 집계 (lite v2) `10점`

**사용 데이터**: Q1에서 불러온 DataFrame (또는 CSV 로드). **MySQL 사용 금지**, Pandas만.

**최종 출력 기대값**: shape `(18, 4)`, 첫 행 `WATER / 108 / "13.97%" / Top3`, Top3 = WATER·NORMAL·GRASS

| 체크 항목 | 기준 | 배점 |
|---|---|:---:|
| `Type` groupby → count | `groupby('Type').size()` 또는 `.count()` | 2 |
| 컬럼명 `count` | `Number_of_Pokemons` 가 아니라 정확히 `count` | 1 |
| `ratio` 컬럼 | 전체 대비 비율, **문자열 `"xx.xx%"` 포맷, 소수 2자리** | 2 |
| `tier_label` 컬럼 | count 상위 3개 → `"Top3"`, 그 외 → `"Others"` | 2 |
| Top3 행 개수 검증 | 정확히 3개 | 1 |
| 정렬 | `count` 내림차순 | 1 |
| 컬럼 순서 | `Type, count, ratio, tier_label` (정확히) | 1 |

**자주 나오는 감점 포인트 (v2 난이도 포인트)**:
- `ratio` 가 float (`0.1397`) 또는 `"13.97"` (%) 기호 누락 → **-2점**
- `tier_label` 경계 오인 (Top3 = 2개 또는 4개) → **-2점**
- 컬럼 순서 뒤바뀜 (`tier_label` 을 `ratio` 앞에) → **-1점**

### Q4. 함수 선언 (삭제 전/후 비교, lite v2) `10점`

**함수 시그니처**: `prep_function(df, drop_col, drop_data)`
**호출 예시**: `result_df = prep_function(df=tmp_df, drop_col='Type', drop_data=['GRASS', 'POISON'])`
**최종 반환 기대값**: shape `(18, 4)`, 컬럼 `Type, before, after, pct_change`. GRASS·POISON 행은 `after=0 / pct_change=-100.0`.

| 체크 항목 | 기준 | 배점 |
|---|---|:---:|
| 함수 선언 | `def prep_function(df, drop_col, drop_data)` 형태 | 1 |
| `before` 집계 | 원본 전체 Type 별 count | 1 |
| `after` 집계 | `drop_data` 값 제외 후 Type 별 count | 2 |
| 제외된 Type 처리 | `after_df` 에서는 포함 여부 무관, **반환 DataFrame 에서는 `after=0`** | 2 |
| `pct_change` 계산 | `(after - before) / before * 100`, 소수 1자리 | 2 |
| `before==0` 예외 | `np.nan` 사용 (inf 금지) | 1 |
| 단일 DataFrame 반환 | 튜플·리스트 아닌 **DataFrame 1개 반환** | 1 |
| 시각화 (별도 셀) | grouped bar, 제목 `"Before vs After by Type"`, 범례 | (Q4 그래프 배점은 별도 없음 — 누락 시 -2 감점) |

**자주 나오는 감점 포인트 (v2 난이도 포인트)**:
- 제외된 Type 행을 반환 DataFrame에서 통째로 빼버림 → **-2점** (요구는 `after=0` 유지)
- `pct_change` 소수 2자리·3자리 → **-1점**
- `before==0` 처리를 0 또는 inf 로 남김 → **-1점**
- 함수가 튜플 반환 (`before_df, after_df, ...`) → **-2점**
- SQL로 삭제 처리 → **-3점** (명시적 위반)

## v2 Q2·Q4 LLM 저항 체크리스트 (강사용)

학생이 Claude Code 같은 LLM 에 그대로 붙여넣어 푼 것으로 의심되는 경우, 아래 특징을 보면 판단 가능합니다.

### Q2 LLM 흔적
- `ratio` 를 float 로 계산해놓고 `.round(2)` 로만 처리 → 포맷 누락
- `tier_label` 를 `if else` 3단 분기로 작성 (본 v2 는 2분류)
- 불필요한 `pd.cut` 이나 `Generation` 같은 없는 컬럼 사용

### Q4 LLM 흔적
- 함수가 3-튜플 반환 / `typing.Tuple` 힌트
- 함수 안에 `print(f"...")` 또는 `display()` 가 있음
- `docstring` 이 Google-style 로 과하게 작성됨 (Args/Returns/Raises)
- 시각화 코드에 `twinx()` 사용 (v2 lite 에서는 불필요)
- 타입 검증 `isinstance` + `raise TypeError(...)` 코드

위 특징이 여러 개 동시에 보이면 LLM 의존 의심 사례로 분류 (감점 근거는 아니고, 개별 체크 항목 위반 여부로 감점).
