## 채점 원칙

- ipynb 셀에 **output이 실제로 렌더링**되어 있어야 O (코드만 있고 output 없으면 X)

- 대소문자 오기·rename 미사용 등은 결과 일치 시 O (관용 기준)

## Q1. 데이터베이스로부터 데이터 불러오기 (10점)

**요구사항**
- MySQL `pokemon_prep` 테이블에 `mysql.connector.connect`로 접속
- `Name`, `Type` 컬럼만 SELECT
- WHERE 필터: `Type IN ('GRASS', 'POISON', 'FIRE', 'FLYING', 'DRAGON')`
- 결과를 DataFrame으로 출력 (output 렌더링)

**O**: 위 요구사항 모두 충족 + DataFrame 출력 셀 표시
**X 사례**: 필터 누락/오타 / output 없음 / `to_sql` 역방향 사용

> v1에서는 DISTINCT가 명시 요구가 아니므로 OR/IN 어느 쪽이든 결과 일치 시 O.

## Q2. 포켓몬 타입별 포켓몬 숫자 및 비율 확인 (10점)

**요구사항** — Pandas만 사용 (MySQL 사용 금지)
- 셀 1 (count 단계): `Type` groupby → 포켓몬 수 집계, 내림차순 정렬, DataFrame 출력
- 셀 2 (ratio 단계): 위 결과에 `ratio` 컬럼 추가 (전체 대비 비율, 자릿수 일관)
- 컬럼명은 자유 (`count` / `Number_of_Pokemons` 등 모두 인정)
- 두 셀 각각 output 렌더링 (참고이미지 `Q2_result1.png`·`Q2_result2.png` 두 단계)

**O**: count 단계 + ratio 추가 단계 둘 다 출력 + 정렬 일치
**X 사례**: `MySQL로 집계` / `정렬 누락` / `ratio 누락` / `output 없음`

> v1에서는 `tier_label` 컬럼 요구가 없고, ratio 포맷도 자유(소수 1~2자리·% 표기 모두 인정).

## Q6. 테이블에 삭제된 데이터 재삽입 (10점)

**요구사항**
- `pokemon_prep.csv`에서 `Type IN ['GRASS','POISON']` 필터

- `conn.commit()` 호출

**O**: placeholder + executemany/iterrows + commit + 전체 컬럼 + 재조회 출력
**X 사례**: `f-string INSERT` (SQL Injection) / `commit 누락` / `컬럼 일부 누락` / `to_sql 사용` (placeholder 미사용) / `새 테이블 생성`
