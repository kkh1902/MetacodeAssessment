## 3. Q4 — "함수 선언" 반환값 계약 도입

### 수정 전 (이전버전)
- 제목: **"Q4. 함수 선언"**
- `prep_function(df, drop_col, drop_data)` 함수를 작성해 특정 컬럼의 특정 값을 삭제한 뒤 삭제 전후를 **시각화**
- 반환값 명세 없음 (시각화만 수행)
- 호출 예시 스캐폴드 제공: `prep_function(df=tmp_df, drop_col='Type', drop_data=['GRASS', 'POISON'])`
- 색상 자유 지정

### 수정 후 (v2)
- 제목: **"Q4. 함수 선언 (삭제 전/후 비교)"**
- 동일 시그니처(`prep_function(df, drop_col, drop_data)`) 유지 + **반환 DataFrame 계약 명시**:
  - `Type` (원본의 모든 Type, 제외 대상도 포함)
  - `before` (삭제 전 count, 정수)
  - `after` (삭제 후 count, 제외 대상은 `0`)
  - `pct_change` — `(after - before) / before * 100`, **소수점 1자리 반올림**
- **시각화 셀 분리** → `# Q4 시각화 (grouped bar chart)` 별도 코드 셀로 독립
- 함수 로직(반환 DataFrame)과 시각화 책임 분리

### 개정 의도
- 이전버전은 "함수가 뭘 리턴해야 하는지"가 모호해서, 학생에 따라 DataFrame / tuple / 아무것도 리턴 안 함(시각화 side-effect만) 이 제각각 → 채점 일관성 문제
- **반환값 계약 4컬럼을 고정 → 함수 단위 채점 가능** (DataFrame 쉐이프·dtype·값 비교로 O/X)
- 시각화 셀 분리로 "함수 작성"과 "시각화"를 각각 독립 평가할 수 있음

### 코드 셀 변경

| | 이전버전 | v2 |
|---|---|---|
| 코드 셀 개수 | **1개** (함수 선언 + 호출 + 시각화 혼재) | **2개** (함수 / 시각화 분리) |
| 셀 1 스캐폴드 | `def prep_function(df, drop_col, drop_data):` + 본문 빈 줄 + 즉시 호출 `prep_function(df=tmp_df, drop_col='Type', drop_data=['GRASS', 'POISON'])` | `# Q4 함수 선언 및 실행` (빈 스캐폴드, 학생이 함수·호출·결과 저장 자유 구성) |
| 셀 2 스캐폴드 | 없음 | `# Q4 시각화 (grouped bar chart)` (별도 셀) |
| 호출 예시 위치 | 코드 셀 내부에 하드코딩 | 마크다운 지문에 예시 표기 (`result_df = prep_function(df=tmp_df, drop_col='Type', drop_data=['GRASS', 'POISON'])`) |
| 출력 이미지 셀 | `참고이미지/Q4/Q4_result3.png` 1장 | **삭제** (계약·차트 스펙으로 대체) |

함수 선언 셀에서 호출부를 제거해 **학생이 반환값을 변수로 받아 다음 셀에서 시각화에 넘기는 파이프라인**을 스스로 구성하도록 유도. 이전버전은 함수 정의 직후 같은 셀에서 호출하면서 반환값을 버려서, 시각화가 함수 내부에 있는지 외부에 있는지 구분이 안 됐음.

### 피드백 요청

3. 시각화를 grouped bar chart로 못박았는데, 학생이 stacked bar나 side-by-side 다른 형태로 그렸을 때 감점할지/허용할지.
4. 호출 예시를 코드 셀에서 마크다운으로 뺐는데, 학생이 호출부를 빼먹고 함수 정의만 제출할 위험이 있습니다. 코드 셀에 호출 라인만 주석 처리해서 남겨둘지?
