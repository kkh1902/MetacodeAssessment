## Q1. 클래스 생성자 (10점)

**요구사항**
- `__init__(df, id_col)` 구현
- 객체 속성으로 DataFrame, id 컬럼명 저장
- 객체 생성 시 데이터 정보 자동 출력

## Q2. 데이터 변형 (Melt → Long Form) (10점)

## Q9. Git Step1 & Step2 (Pandas 모델 준비) (10점)

**요구사항**
- **main 브랜치**에서:
  - Step1: 라이브러리 import + data 로드
  - Commit 메시지 "Step1"
- **prep_branch 생성 후 체크아웃**:
  - Step2: train_test_split (`random_state=42`), LabelEncoding
  - Commit 메시지 "Step2"

**O**: 브랜치 이름 정확 + commit 메시지 정확 + random_state=42
**X**: 브랜치명 다름 / commit 메시지 오타 / random_state 누락

---

## Q10. Git Step3 + Merge (모델 학습) (10점)

**요구사항**
- **fin_branch 생성 후 체크아웃**:
  - Step3: `ExplainableBoostingClassifier(interactions=False)` 학습
  - `accuracy_score`, `classification_report` 출력
  - Commit 메시지 "Step3"
- **main에 merge** (분기가 그래프에 보이도록)
  - fast-forward 시 분기 안 보임 → `--no-ff` 또는 병행 커밋으로 분기 유지

**O**: EBC 모델 학습 + 정확도 출력 + main으로 merge + 분기 그래프 유지
**X**: main으로 merge 누락 / EBC 미사용 / fast-forward로 분기 사라짐

> **주의**: 정확도·리포트 수치가 답지와 완전 동일하지 않아도 O. `random_state=42`로 split 재현성만 확보되면 됨 (EBC 내부 stochastic 요소 허용)
