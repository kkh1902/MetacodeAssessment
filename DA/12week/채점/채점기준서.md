## Q1. 클래스 설계 — 네이버 뉴스 크롤링 (10점)

**요구사항**
- 클래스 `news_collector`
- 메서드 `naver_news_search(search_keyword_list)`:
  - 검색어별 뉴스탭 첫 화면 기사 수집 (스크롤 금지)
  - 수집 필드: 검색어(keyword), 제목, 언론사, 내용 (참고이미지 Q1_3·Q1_4 기준 4컬럼)
  - **중복 제거 기준**: 제목+언론사 동일 시 1건
  - `self.result_df`에 DataFrame 저장
  - 상위 5개 언론사를 `self.press_top5`에 저장
- 메서드 `press_search(press_col, keyword_col, press_list)`:
  - 언론사별 기사 비율(%) 계산
  - **Donut chart 출력** (pie + 중앙 구멍 옵션 필수)

**O**: 클래스 + 두 메서드 + 중복 제거 + `self.result_df` / `self.press_top5` 속성 + donut chart
**X**: requests/Selenium 크롤링 미동작 / donut 옵션 없음 / 속성명 다름 (※ 중복 제거는 결과 일치 시 관용)

## Q2. Pandas 전처리 + 시각화 (10점)

**요구사항** (참고이미지 Q2_1_1 / Q2_3 기준)
- **전처리 1**: `orders.csv`(status='Shipped' 필터) + `users.csv` 조인 → 3회 이상 주문자만 추출 후 **국가별 점유율 계산**
  - `user_count` = 국가별 3회+ 주문자 수
  - `user_ratio` = `user_count / SUM(user_count) × 100` (전체 568명 중 점유율, 소수 1자리)
- **전처리 2**: 국가별 주문 횟수 비율 (status='Shipped'), `order_count` + `order_ratio = order_count / SUM × 100`
- **데이터 결합**: 위 두 DataFrame을 country로 merge → 5컬럼 결과
- **시각화 1**: `user_count` Pie chart
- **시각화 2**: `order_count` Pie chart

**O**: status 필터 + 두 전처리 (점유율 방식) + merge + 두 pie chart
**X**: status 필터 누락 / user_ratio가 점유율(전체 대비 국가 분포)이 아닌 다른 정의 / pie chart 누락
