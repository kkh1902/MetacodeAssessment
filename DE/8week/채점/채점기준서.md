# DE 8주차 채점 기준서 (260429-version)

## 채점 원칙

- **0/10 이진 채점** (각 문제 10점 만점, 부분 점수 없음)
- 7문제 × 10점 = **70점 만점** (Q1~Q5 + Q9 + Q10)
- 핵심 요구사항 모두 충족 → 10점 / 하나라도 미흡 → 0점
- **양식(폴더 구조·파일명) 미준수 시 −10점 추가 감점**
- **PDF 단독·docx 단독·코드 이미지 단독 제출 → 전체 0점**

## 자동 채점 흐름

각 문제는 **2단계**로 채점한다.

| 단계 | 내용 |
|------|------|
| ① 파일 존재 확인 | `Q{N}/` 폴더 안에 명시된 파일이 모두 있는지 |
| ② 코드 키워드 grep | `.py` / `.yaml` / `.sh` 안에 필수 키워드 포함 여부 |

자동 채점 통과 후 **③ 캡처 수동 확인** (캡처 내용 + 본인 증빙).

## Q9 — Self-Healing (10점)

### 제출물
- `Q9/nginx-deployment.yaml`
- `Q9/self_heal_delete.png`
- `Q9/self_heal_kill.png`
- `Q9/ec2_verify.png`

### 자동 채점 키워드 (`nginx-deployment.yaml`)
- `livenessProbe`
- `replicas: 3`
- `kind: Deployment`

### 채점자 환경 재현 검증
```bash
kubectl apply -f 학생_Q9/nginx-deployment.yaml
kubectl get pods -l app=nginx-self-heal      # 3개 Running 확인
POD=$(kubectl get pods -l app=nginx-self-heal -o jsonpath='{.items[0].metadata.name}')
kubectl delete pod $POD
sleep 10
kubectl get pods -l app=nginx-self-heal      # 새 Pod 자동 생성 확인 → ✓
kubectl delete -f 학생_Q9/nginx-deployment.yaml
```

### 캡처 수동 확인
- `self_heal_delete.png`: Pod 삭제 → 재생성 시간차 (T+0, T+30s 권장)
- `self_heal_kill.png`: `RESTARTS` 컬럼 0 → 1+ 증가 캡처
- `ec2_verify.png`: 본인 IAM ARN + 인스턴스 ID + `kubectl get nodes -o wide` 한 화면

### X 판정 사례
- livenessProbe 누락
- replicas 3개 아님
- 캡처가 정지된 한 시점만 (시간차 없음)
- ec2_verify.png 의 IAM ARN 사용자명이 본인 이름 아님

## Q10 — Rolling Update (10점)

### 제출물
- `Q10/nginx-deployment.yaml`
- `Q10/load_test.sh`
- `Q10/curl_log.txt`
- `Q10/rolling_update.png`
- `Q10/rollout_history.png`
- `Q10/ec2_verify.png`

### 자동 채점 키워드
- `nginx-deployment.yaml`:
  - `strategy:` 와 `rollingUpdate:`
  - `maxSurge`
  - `maxUnavailable`
  - `replicas: 3`
- `load_test.sh`:
  - `curl` 호출
  - `while true` 또는 무한 루프
- `curl_log.txt`:
  - **200 외 응답 0건**

### 채점자 환경 재현 검증
```bash
kubectl apply -f 학생_Q10/nginx-deployment.yaml
chmod +x 학생_Q10/load_test.sh
./학생_Q10/load_test.sh > /tmp/curl_log.txt 2>&1 &
LOAD_PID=$!
sleep 5
kubectl set image deployment/nginx-rolling nginx=nginx:1.25
kubectl rollout status deployment/nginx-rolling
kill $LOAD_PID
grep -v ' 200$' /tmp/curl_log.txt | grep -E '[0-9]{3}$' | wc -l   # 0이면 ✓
kubectl rollout history deployment/nginx-rolling   # revision 2개 이상
kubectl delete -f 학생_Q10/nginx-deployment.yaml
```

### 캡처 수동 확인
- `rolling_update.png`: `kubectl rollout status` 진행 화면
- `rollout_history.png`: revision 2개 이상 보이는 history 출력
- `ec2_verify.png`: Q9와 동일

### X 판정 사례
- `maxUnavailable: 0` 아님 (다운타임 허용)
- `curl_log.txt` 에 5xx/000/error 응답 1개라도 있음
- rollout_history 에 revision 1개만
- 캡처 누락

## 채점 결과 작성 (`채점결과.xlsx`)

- 행: Q1, Q2, Q3, Q4, Q5, Q9, Q10, 양식준수, 총점, 복붙용 피드백
- O 셀: `O` (10점)
- X 셀: 짧은 감점 사유 (예: `livenessProbe 누락`, `maxUnavailable=0 아님`, `curl_log 5xx 발생`)
- 양식준수 행: 준수=O, 미준수=−10점 사유 명시
- 총점: O 합계 × 10 − 양식 감점
- 복붙용 피드백:
  - 만점: `70\n(감점 없음)`
  - 감점: `{총점}\n{Q번호}. {사유}\n{Q번호}. {사유}...`
