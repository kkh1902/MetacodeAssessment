## 채점 방식
- 각 문제별 배점: **10점**
- O (10점): 핵심 요구사항 충족
- X (0점): 미충족 또는 미제출
- 총점: **100점** (10문제)
- 문제 9 (fetch_subway_data): 수집(Task1) + 분석(Task2) **모두** 충족해야 O

## 채점 시 주의사항

1. **파일명이 다르더라도** 동일한 기능을 구현했으면 해당 문제로 인정
2. **S3Hook vs BaseHook**: 둘 다 Airflow Connection 활용이므로 O 부여
3. **AWS 키 하드코딩**: AWS 키를 코드에 직접 넣는 것은 감점 사유 아님 (단, Airflow Connection 활용은 필수)
4. **스크린샷 제출**: K8s 과제(9, 10)는 스크린샷도 인정, 그 외 과제는 코드 필수
5. **정답과 다른 접근이라도** 요구사항을 모두 충족하면 O
6. **수강생 제출 원본 파일은 절대 수정/삭제하지 말 것**
8. **실행 스크린샷/로그 제출 필수**: 제출물에 "실행 스크린샷 또는 실행 로그"가 명시된 과제는 코드만 제출 시 X 처리
   - 스크린샷/로그 필요 과제: 1-1, 1-2, 2-1, 3, 4, 5, 6, 9/10
   - 코드만 제출하면 되는 과제: 2-2, 7/8
7. **S3 관련 과제(과제2, 과제7, 과제8) 검증**: 실제 AWS S3에 접속할 수 없으므로, **LocalStack(가짜 S3)**을 로컬에 띄워서 학생 코드를 실행하여 동작 여부를 확인한다

## 엑셀 채점 결과 출력 형식

- 파일: `8week/8주차_채점결과.xlsx`
- 시트명: `8주차 채점`
- 구조:
  - 행: 수강생 (이름 가나다순)
  - 열: No, 과제명, 설명, 배점, 수강생1, 수강생2, ...
  - 셀 값:
    - O인 경우: `O`
    - X인 경우: 틀린 이유 (예: `X - window 함수 기반 집계 누락`)
  - 합계 행: 각 수강생의 총점
  - 스타일:
    - O 셀: 초록 배경
    - X 셀: 빨강 배경
    - 헤더: 파랑 배경 + 흰색 글씨
    - 합계: 노랑 배경 + 굵은 글씨
