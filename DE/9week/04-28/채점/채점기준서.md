## 채점 시 주의사항

1. **파일명이 다르더라도** 동일한 기능을 구현했으면 인정
2. **스크린샷 필수 문제**: 2, 3, 4, 5, 6, 7, 8, 9, 10번 → 코드만 제출 시 X
3. **코드만 제출 가능 문제**: 1번 (click_producer.py)
4. **approx_count_distinct**: 문제 5에서 `countDistinct` 대신 사용해도 O 인정
5. **BashOperator로 spark-submit**: 문제 10에서는 `SparkSubmitOperator` 명시 요구이므로 X
6. **수강생 제출 원본 파일은 절대 수정/삭제하지 말 것**
7. **정답과 다른 접근이라도** 요구사항을 모두 충족하면 O

## 엑셀 채점 결과 출력 형식

- 파일: `9week/9주차_채점결과.xlsx`
- 시트명: `9주차 채점`
- 구조:
  - 행: 수강생 (이름 가나다순)
  - 열: No, 과제명, 설명, 배점, 수강생1, 수강생2, ...
  - 셀 값:
    - O인 경우: `O`
    - X인 경우: 틀린 이유 (예: `X - SparkSubmitOperator 미사용`)
  - 합계 행: 각 수강생의 총점
  - 스타일:
    - O 셀: 초록 배경
    - X 셀: 빨강 배경
    - 헤더: 파랑 배경 + 흰색 글씨
    - 합계: 노랑 배경 + 굵은 글씨
