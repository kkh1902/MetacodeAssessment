# 5주차 과제 채점 기준서

> **채점 원칙: 문제당 0 or 10, 부분점수 없음**

## 채점 방식
- 각 문제별 배점: **10점** (O = 10점 / X = 0점, 이진 채점)
- 총점: **100점** (10문제 × 10점)
- O (10점): 핵심 요구사항 **모두** 충족
- X (0점): 하나라도 미흡 / 미제출

## 제출 형식
- 문서 파일(.docx/.hwp/.rtf/.pages/.pdf)이어도 내용이 평가 가능하면 채점 (5주차는 형식 규칙 미적용)
- 코드를 이미지(캡쳐)로만 제출했을 때만 0점 처리
- S3 연동 키는 `' '`로 비워서 제출 (감점 사유 아님)

## 채점 시 주의사항

1. **파일명이 달라도** 동일 기능 구현 시 O 인정
2. **AWS 키 하드코딩** 감점 사유 아님 (문제지에서 `' '`로 비워달라고 안내)
3. **스크린샷 형식**은 png/jpg/캡쳐가 ipynb/docx에 삽입되어도 인정
4. **ipynb 출력 셀**로 실행 결과가 보존되어 있으면 캡쳐 대체 인정
5. **정답과 다른 접근이라도** 요구사항을 모두 충족하면 O
6. **수강생 제출 원본 파일은 절대 수정/삭제하지 말 것**
7. **Databricks Notebook 제출**도 코드 파일(.ipynb/.py)로 인정
8. **문서 파일(.docx/.hwp/.rtf/.pages) 단독 제출** → 문제지 명시에 따라 전과제 0점

## 엑셀 채점 결과 출력 형식

- 파일: `5week/채점결과.xlsx`
- 시트명: `DE 5주차 채점`
- 구조:
  - 행: 문제 (Q1~Q10) + 총점 + 복붙용 피드백
  - 열: 문제, 학생1, 학생2, ...
  - 셀 값:
    - O인 경우: `O`
    - X인 경우: 감점 사유 텍스트
  - 총점 행: 각 수강생의 총점 (0 or 10 합산)
  - 복붙용 피드백 행: 학생별 총점 + 틀린 문제번호/사유 나열
  - 스타일:
    - O 셀: 초록 배경(C6EFCE) / 초록 글씨(006100) / 가운데 정렬
    - X 셀(감점 사유): 빨강 배경(FFC7CE) / 빨강 글씨(9C0006) / 좌측 정렬
    - 헤더: 파랑 배경(4F81BD) + 흰색 글씨
    - 총점: 진한파랑 배경(305496) + 흰색 글씨
    - 피드백: 연노랑 배경(FFF2CC)
