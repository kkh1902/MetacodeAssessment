# 실전 장애대응 문제 생성

$0주차 문제지와 채점기준서를 생성합니다.

## 사전 참조

반드시 아래 파일들을 먼저 읽고 패턴을 따를 것:

1. 기존 문제지 예시: `11week/문제지/11주차_실전_장애대응_문제지.md`
2. 기존 채점기준서 예시: `11week/채점기준서.md`
3. 버그코드 가이드 (있는 경우): `$0week/참조_src/버그코드_제작_가이드.md`

## 출력 파일

```
$0week/문제지/$0주차_실전_장애대응_문제지.md
$0week/채점기준서.md
```

## AI 차단 문제 설계 원칙 (필수 준수)

### 절대 금지 유형
- CRUD 구현, API 만들기, SQL 작성, Spark 코드 작성, 알고리즘 문제
- "구현하시오", "작성하시오", "해결하시오" 같은 정답형 문제

### 반드시 따를 공식

```
문제 = [깨진 환경] + [로그 제공] + [데이터 이상] + [가설 요구] + [검증 요구] + [전/후 비교] + [재실행 요구]
```

### 환경 의존성 강제
- Docker / 네트워크 / Kafka partition / Spark executor / DB 상태 / cluster 환경 반드시 포함
- 동일 코드인데 결과 다르게 만들기

### 가설-검증 구조 강제
모든 문제에서 반드시 요구:
1. 원인 가설 2개 이상
2. 각 가설 검증 방법
3. 실제 원인 특정
4. 수정 전/후 비교
5. 재실행 결과 증명

### 데이터 트랩 포함 (최소 1개)
- timestamp 1초 차이, key 중복, schema 변경
- irrelevant 로그 섞기, 일부 로그 누락
- staging -> final 누락, join fan-out

### 정답 없는 문제 1개 이상 포함
- KPI 이상 탐지, 데이터 왜곡 원인, 성능 저하 원인

## 문제지 구조

### 공통 섹션
- 과제 안내 (총 문제수, 배점, 마감)
- 공통 요구사항 (5가지 필수 증거)
- 제출 형식 (디렉토리 구조)

### 각 문제 구조
```markdown
# Problem N - 제목

**카테고리**: ... | **배점**: 20점

## 상황 (구체적 장애 시나리오)
## 목표
## 제출물 (표 형태)
## 힌트 (방향만, 답 아님)
```

### 난이도 구조 (5문제 기준)
| 단계 | 역할 |
|------|------|
| Problem 0 | 기본 탈락 필터 (환경) |
| Problem 1 | 개념 필터 (정합성) |
| Problem 2 | 실무 감각 필터 (디버깅) |
| Problem 3 | 기술 깊이 필터 (성능) |
| Problem 4 | 최종 상위 10% 필터 (품질/분석) |

## 채점기준서 구조

### 각 문제별 구조
```markdown
## Problem N: 제목

**카테고리**: ...
**역할**: ...

### 목표
### 상황 설정
### 제출물 (표)

### 자동채점 (12점)
| 항목 | 배점 | 기준 |

### 수동채점 (8점)
| 항목 | 배점 | 기준 |

### 0점 조건
```

### 채점 비율
- 자동채점: 60% (객관적 실행 결과)
- 수동채점: 40% (체크리스트 기반, 주관 배제)

### 버그 코드 제작 원칙
1. 실행은 되어야 함 (import/syntax error 금지)
2. 버그가 자연스러워야 함 (일부러 넣은 티 안 남)
3. 힌트 주석 절대 금지 (TODO, FIXME, NOTE 금지)

### 운영 가이드 포함
- 채점 순서 (자동 -> 수동)
- 수강생 제출 디렉토리 구조
- 엑셀 출력 형식

## 주의사항

- 기존 주차 문제와 중복되지 않도록 할 것
- 카테고리는 데이터 엔지니어링 실무 기반으로 할 것
- 공통 요구사항 5가지(스크린샷, diff, 가설 2개, 원인 특정, 실행 증명) 빠지면 0점 규칙 유지
- "수강생 제출 원본 파일은 절대 수정/삭제하지 말 것" 채점 주의사항 포함
