# AILLM 14주차 채점기준서

> 주제: **LLM Fine-tuning (LoRA) + Multimodal LLM (Mini-LLaVA)**
> 배점: 각 문제 10점 × 10문항 = **100점 만점**

---

## 채점 환경 분배

| 문제 | 채점 환경 | 비고 |
|------|----------|------|
| **Q1, Q2, Q3, Q5** (LoRA Fine-tuning) | **로컬 RTX 4060 8GB** | Llama-3.2-1B-Instruct fp16 + LoRA, batch=2 → VRAM 4~5GB |
| **Q4** (Full FT vs LoRA 비교) | **Kaggle Notebook (T4 16GB) / Colab T4** | Llama-1B fp32 Full FT 모델 + LoRA 동시 보유 → 8GB VRAM 초과 위험 |
| **Q6~Q10** (Multimodal) | **로컬** | CLIP-base + GPT-2, VRAM 1~2GB |

**Q4 채점 전 사전 준비**:
- Kaggle/Colab에서 학생 ipynb 업로드 후 Q4 셀(4-1, 4-2, 4-3)만 재실행 검증
- 또는 학생 제출 출력(`trainable_params`, `total_params`, `out_full`/`out_lora` 텍스트) 그대로 인정
- HuggingFace 토큰 + Llama-3.2-1B-Instruct access 승인 필요

## 문제별 O/X 판정 기준

### Q1. 사전학습 모델 로드 + Baseline 추론 + Instruction 데이터 구성

**Q1-1**: `meta-llama/Llama-3.2-1B-Instruct` 모델/토크나이저 로드, `pad_token = eos_token`
**Q1-2**: `model.generate()` 호출 + `tokenizer.decode()` 출력
**Q1-3**: alpaca dataset, `format_example()` 함수 정의 (Instruction/Response 포맷)
**Q1-4**: `tokenizer(...)` + `tokenized["labels"] = tokenized["input_ids"].copy()`

**O 판정**: 4개 sub-task 모두 정상 동작 (출력 결과 존재)
**X 사례**: 모델 명을 임의 변경 (예: gpt2 사용), labels 누락, format 함수 누락

### Q2. LoRA 설정 + Trainer 구성 + Fine-tuning

**Q2-1**: `LoraConfig(r=8, target_modules=["q_proj","v_proj"], lora_dropout=...)` — **모델 명시 q_proj/v_proj 필수**
**Q2-2**: `get_peft_model(...)` + `print_trainable_parameters()` → `trainable params: 851,968`
**Q2-3**: `TrainingArguments(per_device_train_batch_size=2, num_train_epochs=1, save_strategy="no", report_to="none")` + `Trainer` 객체
**Q2-4**: `trainer.train()` 실행 (loss 표시 진행 바 출력)

**O 판정**: 4개 sub-task 모두 충족, 851,968 trainable params 출력
**X 사례**: target_modules가 `q_proj, v_proj` 외 다른 값 (예: c_attn — gpt2용), batch size 다름, save_strategy 누락

### Q3. Fine-tuning 결과 추론 / 저장 / 재로드

**Q3-1**: `lora_model.generate()` → 응답 텍스트 출력
**Q3-2**: `lora_model.save_pretrained(save_path)`
**Q3-3**: `AutoModelForCausalLM.from_pretrained(model_name)` + `PeftModel.from_pretrained(base, save_path)` + `generate()`

**O 판정**: 3개 sub-task 모두 동작, LoRA 적용된 응답 출력 확인
**X 사례**: save_pretrained 누락, 재로드 시 base + adapter 결합 누락

### Q4. Full FT vs LoRA 비교 ⚠️ **Kaggle/Colab 채점**

**Q4-1**: `full_ft_model = AutoModelForCausalLM.from_pretrained(model_name)` + `total_params`, `trainable_params` 계산 → 둘 다 1,235,814,400 출력
**Q4-2**: `lora_total = sum(p.numel() ...)`, `lora_trainable = sum(... if p.requires_grad)` 계산
**Q4-3**: `full_ft_model.generate(...)`, `lora_model.generate(...)` 동일 프롬프트 비교

**O 판정**: 3개 sub-task 모두 정상 출력 (수치 / 텍스트)
**X 사례**: 변수명 미정의, generate 호출 누락, params 계산 누락

> 로컬 채점 불가 사유: Llama-1B fp32 Full FT 모델(약 5GB) + LoRA 모델(약 2.5GB) 동시 보유 시 RTX 4060 8GB OOM 위험. **Kaggle T4 16GB / Colab T4** 사용.

### Q5. Multi-LoRA Adapter

**Q5-1**: `lora_config_A` + `model_A = get_peft_model(...)` (변수명 `model_A` 필수)
**Q5-2**: `lora_config_B` + `model_B = get_peft_model(...)` (또는 `add_adapter` 방식 인정)
**Q5-3**: `model_A.generate()`, `model_B.generate()` 또는 `set_adapter()` 교체 후 generate

**O 판정**: A/B 두 어댑터 객체로 generate 수행
**X 사례**: model_A / model_B 변수 미정의, lora_config 1개만 정의

> 답지가 r/dropout으로 차별화한 것은 **참조용**. 문제지에는 차별화 강제 조건 없음 → A=B 동일 설정이라도 별도 객체 생성 시 O.

### Q6. CLIP Vision Encoder + GPT2 Tokenizer

**필수**:
1. `processor(images=image, return_tensors="pt")` 변환
2. `vision_model(**inputs)` forward → `last_hidden_state`
3. `tokenizer.pad_token = tokenizer.eos_token` 설정
4. `tokenizer(text, return_tensors="pt", ...)` 적용

**O 판정**: vision embedding shape `[1, 50, 768]` + tokenizer input_ids shape 출력
**X 사례**: pad_token 미설정, processor 미사용

### Q7. Vision → Language Projection

**필수**: `nn.Linear(vision_dim, language_dim)`
**O 판정**: `torch.Size([1, 10, 768])` 출력
**X 사례**: nn.Linear 미사용 (예: nn.Conv 등 다른 layer)

### Q8. Vision/Text Embedding Concat

**필수**: `torch.cat([vision_embeds, text_embeds], dim=1)`
**O 판정**: `torch.Size([1, 16, 768])` 출력
**X 사례**: dim=0 또는 dim=2 사용

### Q9. MiniLLaVAModel.forward() 구현

**필수 4단계**:
1. `vision_outputs = self.vision_model(pixel_values=pixel_values)`
2. `vision_embeds = self.proj(vision_outputs.last_hidden_state)`
3. `text_embeds = self.lm.get_input_embeddings()(input_ids)` (또는 `self.lm.transformer.wte(input_ids)` for GPT2)
4. `inputs_embeds = torch.cat([vision_embeds, text_embeds], dim=1)`

**O 판정**: 4단계 모두 구현, return self.lm(inputs_embeds=..., labels=...)
**X 사례**: projection 누락, concat 잘못된 dim, input_ids로 lm 호출 (inputs_embeds 미사용)

### Q10. Training Step (Vision Encoder Freeze)

**필수**:
1. `for p in model.vision_model.parameters(): p.requires_grad = False` (Vision freeze)
2. `outputs = model(pixel_values=..., input_ids=..., labels=...)` forward
3. `loss = outputs.loss`
4. `loss.backward()`
5. `optimizer.step()`

**O 판정**: 5단계 모두 실행, "Training step completed. Loss: ..." 출력
**X 사례**: freeze 누락, loss.backward() 누락, optimizer.step() 누락

## 채점 원칙

- **O/X 이진 채점**: 부분 점수 없음, sub-task 중 하나라도 빠지면 X
- **출력 결과 일치**: 수치 출력(shape, params)은 정답과 동일해야 O
- **코드 다양성 인정**: 출력이 동일하면 다양한 구현 방식 인정 (예: `self.lm.get_input_embeddings()` ↔ `self.lm.transformer.wte`)
- **모델 명시 준수**: Llama-3.2-1B-Instruct로 명시된 문제는 다른 모델 사용 시 X
- **target_modules 명시 준수**: `q_proj, v_proj`로 명시된 문제는 다른 module 사용 시 X (예: gpt2의 c_attn)
