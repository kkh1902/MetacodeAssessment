# 문제지 생성 양식 가이드

## 개요

DE/AIEngineer 등 각 주차 문제지 PDF는 `reportlab` 라이브러리로 생성한다.  
새 주차 문제지를 만들 때는 기존 `generate_v2.py`를 복사해서 내용만 수정한다.

**참고 스크립트**: `DE/8week/문제지/generate_v2.py` (DE 8주차 v2 기준)

---

## 파일 위치 규칙

```
{트랙}/{N}week/문제지/
  ├── generate_v2.py          ← 생성 스크립트 (작업 후 삭제하지 말 것)
  └── {N}주차_문제지_v2.pdf   ← 최종 산출물
```

---

## 1페이지 (표지) 표준 구성

원본 8주차 주관식 문제지 기준으로 표지는 다음 구성을 유지한다.

| 요소 | 내용 |
|------|------|
| 제목 | `{N} 주차 주관식 문제지` |
| 부제 | 해당 주차 기술 스택 (예: `Kafka · Spark · Airflow 데이터 파이프라인 실습`) |
| 0점 처리 안내 (빨간 박스) | .docx/.hwp 단독 제출 / 코드를 이미지로 제출 / PDF 단독 제출 |
| 제출 형식 (회색 박스) | `파일명: 데엔몇기_이름_몇주차.zip` / 예시 |

---

## 2페이지 (제출 폴더 구조)

ZIP 내부 폴더 구조를 `struct_box()` 함수로 표시.  
폴더명은 `Q1`, `Q2` ... `Q{N}` 형식으로 통일.

```
데엔3기_홍길동_8주차/
  ├── Q1/
  │     ├── {파일명}.py
  │     └── {캡처명}.png
  ...
```

---

## 문제 페이지 구성 (Q당 1페이지 원칙)

각 문제는 다음 4개 섹션으로 구성한다.

### ① 문제 헤더 (`q_header`)
```python
q_header(번호, "문제 이름", score=10)
```
- 파란 배경 박스
- `과제 {번호:02d} — {제목}` / `배점: {N}점`

### ② 과제 설명
- `Paragraph(설명문, S["body"])`
- 2~3줄 이내로 간결하게

### ③ 조건 (`S["bullet"]`)
- `Paragraph("• 조건 내용", S["bullet"])` 형식
- 필수 파일명, 사용할 클래스/함수, 출력 형식, 제약사항

### ④ 제출물 박스 (`submit_box`)
```python
submit_box([
    "Q{N}/{파일명}.py",
    "Q{N}/{캡처명}.png  —  캡처 설명",
])
```
- 노란 배경 박스

### ⑤ 자동 채점 기준 박스 (`check_box`)
```python
check_box([
    "필수 import 또는 클래스명",
    "필수 메서드명",
    "{캡처명}.png 파일 존재",
])
```
- 초록 배경 박스
- grep으로 확인 가능한 키워드만 기재

## 자동 채점 스크립트 연계

`채점/채점기준서.md`의 각 문제에 자동 채점 키워드를 명시한다.  
채점 스크립트는 다음 2단계로 판정한다.

1. **파일 존재 확인**: `수강생/{이름}/Q{N}/{파일명}.py`, `{캡처명}.png` 존재 여부
2. **코드 키워드 grep**: 필수 클래스/함수가 `.py` 파일 내에 포함되는지 확인

두 조건 모두 충족 → O (10점), 하나라도 미충족 → X (0점)
